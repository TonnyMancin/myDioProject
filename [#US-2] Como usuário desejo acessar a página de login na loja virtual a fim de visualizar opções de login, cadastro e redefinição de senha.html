 Descrição   
  1. Valor: A autenticação de usuários na loja virtual é importante para que os clientes possam se cadastrar na plataforma com segurança e fazer pedidos. Essa funcionalidade visa unificar fluxo de acesso ao site permitindo que o usuário possua se cadastrar, fazer login ou redefinir a senha na loja virtual.
  2. Narrativa de usuário:
    1. Como cliente
    2. Eu quero acessar a página de login do usuário
    3. Para visualizar opções de login cadastro e redefinição de senha na loja virtual
  3. Requisitos
  • Atores: Clientes.
  • Interface: Ver documento em anexo da UV/UX.
  • Dados:
    • Criação de banco de dados/ tabelas para armazenar informações dos clientes cadastrados.
    • Criação de API para integração com front-and e banco criado.
    • Criação de tela de login para permitir que acesso páginas de login e páginas de cadastro do usuário.
  • Plataforma/Ambientes:
    • Web/ Web Mobile.
    • Devem ser configurados ambientes e pipelines para desenvolvimentos homologação de usuários, homologação QA e produção.
  • Critérios de aceite
    • Usuários devera visualizar formulários para fazer login e entrar na loja virtual.
    • Usuários devera visualizar botão para acessar tela de cadastro do sistema.
    • Usuários devera visualizar botão para acessar tela “esqueci a senha“.

OU

  1. Cenário: Cliente sem cadastro deseja cria uma conta
    1. Dado que o cliente esteja na tela de login
    2. E não esteja cadastrado no sistema
    3. Quando clicar em “criar conta“
    4. Então será redirecionado para uma tela de criação de uma nova conta.
  2. Cenário: Cliente sem cadastro tenta fazer login
    1. Dado que o cliente esteja na tela de login
    2. E adicione um username não cadastrado
    3. Quando clicar em “criar conta “
    4. Então uma mensagem surgirá avisando que a conta não existe
  3. Cenário: Cliente com cadastro informa dados incorretos de acesso
    1. Dado que o cliente esteja na tela de login
    2. E adicione um usename correto
    3. Mas escreve a senha incorreta
